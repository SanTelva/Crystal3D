CIGOR

Пусть вас не сбивает с толку название формата — с форматом IGOR он имеет мало общего. CIGOR используется, на данный момент, только в программе на C++, решающей задачу определения порога перколяции. По этой причине он весьма специфичен и сильно под эту задачу заточен.
Первой строкой в файле идёт число тестов, которые должны быть проделаны. Эта информация, как легко догадаться, специфична конкретно для этой программы.
Два числа через пробел на второй строке — размеры решётки в элементарных ячейках. Опять же, эта информация весьма специфична, т.к., в принципе, сам кристалл бесконечен, но в этой программе рассматривается его конечный фрагмент.
На третьей строке также записаны два числа через пробел — это количество нод и бондов соответственно.
В четвёртой и далее строки записаны сами бонды. Тонкость — если во всех остальных форматах нумерация отдельная для каждой ячейки, то в формате CIGOR нумерация нод общая для всей решётки — это возможно из-за конечности кристалла. Бонд представляет из себя два числа — номера нод. Такая система позволяет обойтись без шифтов.
