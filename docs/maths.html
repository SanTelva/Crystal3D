Модуль draw2.maths

Этот модуль несколько оторван от задач, связанных с кристаллами. Он реализует классы Matrix и Vector — как легко догадаться, матрицы и вектора. Для начала — Matrix. Его конструктор таков:
__init__(vect1=[1, 0, 0], vect2=[0, 1, 0], vect3=[0, 0, 1])
Он принимает три строки матрицы. По умолчанию матрица получается единичной. Матрицы можно спокойно складывать, вычитать, умножать на другую матрицу и на вектор (аккуратно! Умножение матриц — не коммутативная операция!), к их элементам можно обращаться с помощью оператора []. У матрицы есть атрибут data, являющийся двумерным массивом с, собственно, матрицей. Длина матрицы, которую возвращает len(matrix) — это длина списка data.
Теперь разберёмся с Vector. У него такой конструктор:
__init__(a=0, b=0, c=0)
Он принимает три значения, т.к. векторы у нас трёхмерные. Эти значения — координаты по x, y и z. Векторы можно свободно складывать, вычитать, умножать на другие векторы и на числа. К их элементам можно обращаться с помощью оператора []. У них есть атрибут data, являющийся списком значений — координат вектора. Также у вектора определён метод .length(), возвращающий длину вектора (не количество измерений, а именно вычисленную длину!). Метод len(vector) (.__len__()) просьба не определять, так как это может ввести в заблуждение.
Кроме классов в этом модуле определено две функции:
rotateEvaluation(axis, angle, matrix) -> matrix — возвращает матрицу matrix повёрнутую на угол angle по оси axis (axis может равняться 'x', 'y' или 'z').
getAngle(vect1, vect2) -> angle — возвращает угол между векторами vect1 и vect2 или None, если хотя бы один из векторов нулевой.
